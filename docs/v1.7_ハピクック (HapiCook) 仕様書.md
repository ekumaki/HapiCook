# **ハピクック (HapiCook) 仕様書**

**バージョン:** 1.7

**改訂日:** 2026/02/09

**ステータス:** ドラフト

## **1\. プロジェクト概要**

### **1.1 目的**

WebやSNSに散在する料理レシピを収集し、AI解析によって統一フォーマット（材料・手順）で管理・閲覧できるアプリ「ハピクック」を開発する。

「自炊の習慣化」を支援し、ユーザーが作った料理写真の記録やAIによる画像生成を通じて、愛着のあるレシピ帳を作成する。

### **1.2 ターゲット**

* **フェーズ1:** 開発者本人（個人利用・実証実験）  
* **フェーズ2:** 自炊を行う一般ユーザー（ストア公開予定）

## **2\. 技術スタック**

* **Frontend:** React Native (Expo)  
  * **Keep Awake:** expo-keep-awake (常時点灯制御用)  
  * **Image Picker:** expo-image-picker  
  * **Web View:** react-native-webview  
  * **Share Extension:** Expo Share Intent  
* **Backend/DB:** Firebase (Firestore, Auth, Storage)  
* **AI:** Google Gemini API (モデル: **Gemini 1.5 Pro**)  
* **Monetization:**  
  * **Subscription:** RevenueCat (課金管理)  
  * **Ads:** Google AdMob (リワード広告)

## **3\. UI/UX 要件**

### **3.1 閲覧画面 (View Mode)**

* **Split View:** デバイスに応じた上下/左右分割レイアウト。  
* **Cooking Mode:** 常時点灯機能。  
* **表示情報:** タイトル、材料、手順、メタデータ（時間・カロリー）。

### **3.2 編集画面 (Edit Mode)**

* **画像管理:** 写真の変更、または**AI画像生成（Premium）**。  
* **メタデータ:** カロリー入力対応。  
* **リスト操作:** 材料・手順の編集・並び替え。

### **3.3 新規登録メニュー**

「＋」ボタンで以下を選択。

1. **写真から:** デバイス内の画像。  
2. **カメラ:** その場で撮影。  
3. **Webリンク:** URL/共有機能からスクショ解析。  
4. **直接入力:** AIを使わず手動登録。

## **4\. 機能要件 (MVP \+ Monetization)**

### **4.1 レシピ登録フロー**

1. **入力:** 画像、カメラ、Webリンクのいずれかを選択。  
2. **制限チェック:**  
   * ユーザーの「AI解析クレジット」を確認。  
   * 不足している場合、\*\*「リワード広告を見て解析」\*\*または「Premium登録」を促すダイアログを表示。  
3. **AI解析:** Gemini 1.5 Proにより情報抽出。  
4. **編集・保存:**  
   * 保存時に「レシピ登録数上限」をチェック（Free版のみ）。

### **4.2 会員システムと制限**

* **Free会員:**  
  * レシピ登録数: 最大50件（習慣化を阻害しないライン）。  
  * AI解析: 月10回まで（以降はリワード広告視聴で+1回/都度回復）。  
  * AI画像生成: 利用不可。  
  * 広告: 一部にバナー広告、解析回復時の動画広告。  
* **Premium会員:**  
  * レシピ登録数: **無制限**。  
  * AI解析: **無制限**。  
  * AI画像生成: **利用可能**（文字だけのレシピを美味しそうな画像に変換）。  
  * 機能: 家族共有・複数端末同期（クラウド同期）。  
  * 広告: 非表示。

  }  
}

## **6\. AI プロンプト設計指針 (Gemini 1.5 Pro)**

### **通常解析用 (System Prompt)**

* **タスク:** 画像からレシピ情報（タイトル、材料、手順、カロリー、時間、タグ）をJSONで抽出。

### **画像生成用 (Image Generation Prompt \- Premium)**

* **タスク:** レシピのタイトルと材料から、完成予想図（写真リアリスティック）を生成するプロンプトを作成。  
* **入力:** レシピテキスト。  
* **出力:** Imagen 3 等へ投げるための英語プロンプト（例: "A professional food photography of \[Title\], ..., 8k resolution, cinematic lighting"）。

## **7\. 収益化戦略 (Business Model)**

### **7.1 モデル**

\*\*「ポジティブな課金」\*\*を目指すフリーミアムモデル。

「ハピクック」を使う楽しさを最大化する機能を有料化する。

### **7.2 有料コンテンツの価値定義**

* **AI画像生成:** 「文字だけのレシピ」は味気ないが、AIで画像を生成することで視覚的な楽しみと「料理したい欲」を刺激する。  
* **無制限の安心感:** 料理好きにとって「登録上限」は最大のストレス。これを解除することでデータベースとしての価値を最大化する。  
* **家族共有:** 買い物や献立の課題を解決する実用的な機能。

### **7.3 無料版の設計意図**

* **登録数50件:** ユーザーがアプリに依存し、習慣化するのに十分な数。ここで離脱させず、ファン化させてから課金へ誘導する。  
* **リワード広告:** AI解析回数が尽きても「広告を見れば使える」という救済措置を残すことで、完全なブロック（利用不可）を防ぐ。
